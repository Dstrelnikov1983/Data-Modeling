Критерии сравнения инструментов

На что обращать внимание при выборе

Зачем нужна система критериев?

На рынке более 30 инструментов для моделирования данных. Без чётких критериев выбор превращается в лотерею.

Стартап

  • Нужно быстро и бесплатно
  • Важна простота
  • Достаточно физической модели

Enterprise

  • Нужна сертификация
  • Все три уровня модели
  • Интеграция с процессами

Госсектор (РФ)

  • Реестр отечественного ПО
  • Импортозамещение
  • Сертификация ФСТЭК

Критерий 1. Уровни моделирования

Концептуальный
Сущности и связи
→
Логический
Атрибуты, типы, ключи
→
Физический
DDL, индексы, СУБД
Уровень Что моделируем Кому нужен
Концептуальный Сущности и связи без деталей Бизнес-аналитики, заказчики
Логический Атрибуты, типы, ключи, нормализация Архитекторы данных
Физический DDL, индексы, партиции, специфика СУБД DBA, разработчики
Важно: Некоторые инструменты (dbdiagram.io, pgModeler) работают только на физическом уровне, а enterprise-решения (ERwin, Visual Paradigm) поддерживают все три с трассировкой между ними.

Критерий 3. Forward / Reverse Engineering

Forward Engineering

Модель → DDL-скрипт → БД

  • Генерация CREATE TABLE
  • Генерация ALTER (миграции)
  • Учёт диалекта СУБД
Проектирование

Reverse Engineering

БД → визуальная модель

  • Подключение к live-БД
  • Импорт DDL-файла
  • Автоматические связи по FK
Документирование

Round-trip

Модель ↔ БД (в обе стороны)

  • Синхронизация изменений
  • Schema diff + миграции
  • Модель всегда актуальна
Полный цикл

Ещё ключевые критерии

4. Поддержка СУБД

  • Количество СУБД
  • Генерация DDL с учётом диалекта
  • Live-подключение к БД
  • NoSQL (MongoDB, Cassandra)

5. Совместная работа

  • Real-time редактирование
  • Комментарии и рецензии
  • Управление доступом
  • Шаринг по ссылке

6. Версионирование

  • История версий модели
  • Model diff (сравнение)
  • Интеграция с Git
  • Генерация миграций (ALTER)

7. Формат описания

  • Визуальный (drag & drop)
  • Текстовый DSL (DBML, PlantUML)
  • Комбинированный
  • Git-friendly для code review

Критерии выбора (продолжение)

8. Импорт / Экспорт

  • SQL (DDL), PDF, PNG/SVG
  • JSON, XML, DBML
  • Импорт из других инструментов
  • Data Dictionary (HTML/PDF)

9. Документирование

  • Словарь данных (Data Dictionary)
  • Описания таблиц и колонок
  • Бизнес-глоссарий
  • HTML/PDF-отчёты

10. Стоимость

  • Бесплатно / Open Source
  • Freemium (базовый бесплатно)
  • Подписка (per user / month)
  • Enterprise (тысячи $)

11. Порог входа

  • Время до первой диаграммы
  • Документация и обучение
  • Интуитивность интерфейса
  • Сообщество и поддержка

Критерии для enterprise

12. Интеграции

  • CI/CD (генерация DDL)
  • Confluence / Wiki
  • Jira / трекеры задач
  • IDE (DataGrip, VS Code)

13. Масштабируемость

  • Макс. кол-во таблиц
  • Производительность (100+ таблиц)
  • Subject Areas (подмодели)
  • Несколько схем

14. Генерация тестовых данных

  • Random data по паттернам
  • Reverse regex
  • Ссылочная целостность
  • Экспорт в SQL INSERT

Часть 4

Российские продукты

Импортозамещение в моделировании данных

Контекст: импортозамещение

С 2022 года госсектор и объекты КИИ обязаны переходить на отечественное ПО, включённое в Реестр российского ПО Минцифры.

Что уже есть в РФ

  • СУБД: Postgres Pro, Arenadata DB, YDB, Tarantool
  • BI: Visiology, DataLens, Polymatica
  • ETL/MDM: DATAREON Platform

Проблемная ниша

  • Нет прямого аналога ERwin или pgModeler в реестре
  • SILA Union — ближайший, но это аналог ARIS, а не ERwin
  • На практике используют Open Source

DbSchema и Visual Paradigm

Подробный разбор двух универсальных инструментов

DbSchema: подробный разбор

Визуальное проектирование

  • Drag & drop создание таблиц
  • Несколько диаграмм для одной схемы (фокус на подсистемах)
  • Virtual Foreign Keys для БД без FK
  • Подсветка связей при выделении

Schema Sync

  • Сравнение модели и live-БД
  • Визуальный diff изменений
  • Автогенерация ALTER-скриптов
  • Поддержка нескольких окружений (dev/staging/prod)

Random Data Generator

  • Генерация данных по паттернам
  • Reverse regex для строк
  • Соблюдение FK-связей
  • Экспорт в SQL INSERT
70+ СУБД: PostgreSQL, MySQL, Oracle, SQL Server, MongoDB, Cassandra, Redshift, Snowflake, ClickHouse, SQLite, MariaDB, Firebird, DB2 и др.

Visual Paradigm: подробный разбор

Три уровня ERD

  • Концептуальный → Логический → Физический

  • UML 2.5 (все 14 типов диаграмм)

Интеграция моделей

  • ERD ↔ UML Class Diagram
  • ERD → ORM mapping (Hibernate)
  • BPMN ↔ требования
  • Единый репозиторий проекта

Групповая работа: выбор инструмента

Разделитесь на 3 группы. Каждая группа — команда на конкретном предприятии. Выберите и обоснуйте набор инструментов (10 мин).

1 Стартап (5 человек)
Разрабатываете SaaS-продукт. PostgreSQL + MongoDB. Бюджет на инструменты — $0. Нужно: модель БД, документация, совместная работа.
2 Госкорпорация (50 человек)
Объект КИИ. Обязательно реестр отечественного ПО. Oracle → Postgres Pro миграция. Нужно: reverse engineering Oracle, forward engineering PostgreSQL, Data Dictionary.
3 Промышленное предприятие «Руда+»
MES-система. PostgreSQL в Yandex Cloud. Команда 8 человек (3 аналитика + 5 разработчиков). Нужно: концептуальная модель для бизнеса, физическая для разработки, BI-дашборды.

Резюме

14 критериев сравнения

  • Уровни моделирования
  • Forward / Reverse / Round-trip
  • Нотации, СУБД, совместная работа
  • Стоимость и порог входа

8 инструментов

  • Лёгкие: dbdiagram.io, draw.io
  • Специализированные: pgModeler, DbSchema
  • IDE: DBeaver, DataGrip
  • Enterprise: ERwin, Visual Paradigm

Российский контекст

  • Нет прямого аналога ERwin в реестре
  • SILA Union — ближайший (архитектура)
  • Open Source — безопасная альтернатива
  • DbSchema / VP — для коммерческого сектора
